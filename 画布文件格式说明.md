## 项目结构

画布系统由以下核心文件组成：

```
src/
├── types/
│   ├── project.ts      # 项目和画布类型定义
│   └── context.ts      # 上下文类型定义
├── stores/
│   └── projectStore.ts # 项目状态管理
├── components/
│   ├── InfiniteCanvas.vue  # 无限画布组件
│   └── VoiceNote.vue       # 语音/文本笔记组件
├── views/
│   └── CanvasView.vue      # 画布页面视图
└── composables/
    ├── useVoiceRecorder.ts # 录音功能
    ├── useFunASR.ts        # 语音识别
    └── useQwenAgent.ts     # AI 对话
```

## 核心类型定义

### 1. 项目 (Project)

```typescript
interface Project {
  id: string                    // 项目唯一标识
  name: string                  // 项目名称
  createdAt: number             // 创建时间戳
  updatedAt: number             // 更新时间戳
  canvases?: CanvasPage[]       // 画布页数组（多页支持）
  currentCanvasIndex?: number   // 当前画布页索引
  context?: ProjectContext      // 关联的上下文
}
```

### 2. 画布页 (CanvasPage)

```typescript
interface CanvasPage {
  id: string                    // 画布页唯一标识
  type: 'infinite' | 'pdf'      // 画布类型（无限画布或PDF）
  viewport: Viewport            // 视口状态
  nodes: CanvasNode[]           // 节点数组
  createdAt: number             // 创建时间戳
}
```

### 3. 视口 (Viewport)

```typescript
interface Viewport {
  x: number                     // X轴偏移量（像素）
  y: number                     // Y轴偏移量（像素）
  zoom: number                  // 缩放比例（0.1 - 2.0）
}
```

## 画布数据结构

### 多页画布系统

画布采用多页设计，支持以下特性：

- **自动分页**：当当前页有内容时，可以添加新页
- **空页清理**：自动删除空白页面（保留当前页）
- **页码导航**：支持上一页/下一页切换
- **独立视口**：每页拥有独立的视口状态

### 画布页管理

```typescript
// 创建新画布页
function createCanvasPage(id?: string): CanvasPage {
  return {
    id: id || `canvas-${Date.now()}`,
    type: 'infinite',
    viewport: { x: 0, y: 0, zoom: 1 },
    nodes: [],
    createdAt: Date.now()
  }
}
```

### 数据迁移

旧版本使用单画布（`canvas` 字段），新版本自动迁移到多画布（`canvases` 数组）：

```typescript
function migrateProject(project: Project): Project {
  if (project.canvases && project.canvases.length > 0) {
    return project  // 已迁移
  }
  
  if (project.canvas) {
    project.canvases = [{ ...project.canvas, createdAt: project.createdAt }]
    project.currentCanvasIndex = 0
  } else {
    project.canvases = [createCanvasPage()]
    project.currentCanvasIndex = 0
  }
  
  return project
}
```

## 节点类型

### 画布节点 (CanvasNode)

```typescript
interface CanvasNode {
  id: string                    // 节点唯一标识
  type: 'voice-note' | 'text-note'  // 节点类型
  position: { x: number; y: number }  // 节点位置（画布坐标）
  
  // 语音笔记特有
  audioPath?: string            // 音频文件路径
  duration?: number             // 录音时长（毫秒）
  
  // 转写相关
  transcript: string | null     // 转写文本
  transcriptStatus: 'pending' | 'processing' | 'done' | 'error'
  
  // AI 回答相关
  agentResult: string | null    // AI 回答内容
  agentStatus: 'pending' | 'processing' | 'done' | 'error'
  
  // 状态标记
  selectedAsContext?: boolean   // 是否被选为上下文
  isFavorite?: boolean          // 收藏状态
  
  createdAt: number             // 创建时间戳
}
```

### 节点类型说明

#### 语音笔记 (voice-note)

- **创建方式**：长按画布空白处录音
- **音频存储**：`audio/{nodeId}.{extension}`（webm 或 wav）
- **自动转写**：录音完成后自动调用 FunASR 进行语音识别
- **自动 AI 回答**：转写完成后自动提交给大模型生成回答

#### 文本笔记 (text-note)

- **创建方式**：双击画布空白处
- **编辑模式**：创建后直接进入编辑状态
- **支持 Markdown**：文本内容支持 Markdown 格式

### 节点状态流转

```
创建节点
    │
    ├─→ 语音笔记 ──→ 录音完成 ──→ 转写中 (processing)
    │                              │
    │                              ├─→ 转写成功 (done) ──→ AI 处理中 (processing)
    │                              │                         │
    │                              │                         ├─→ AI 回答完成 (done)
    │                              │                         └─→ AI 回答失败 (error)
    │                              │
    │                              └─→ 转写失败 (error)
    │
    └─→ 文本笔记 ──→ 编辑完成 ──→ AI 处理中 (processing)
                                   │
                                   ├─→ AI 回答完成 (done)
                                   └─→ AI 回答失败 (error)
```

## 上下文系统

### 上下文文件 (ContextFile)

```typescript
interface ContextFile {
  id: string                    // 文件唯一标识
  name: string                  // 文件名称
  type: 'static' | 'dynamic'    // 上下文类型
  color: string                 // 标签颜色（预设8种）
  content: string               // 内容（Markdown 格式）
  createdAt: number             // 创建时间戳
  updatedAt: number             // 更新时间戳
  projectId?: string            // 动态上下文关联的项目ID
}
```

### 预设颜色

```typescript
const CONTEXT_COLORS = [
  '#66bb6a', // 绿色
  '#4299e1', // 蓝色
  '#ed8936', // 橙色
  '#e53e3e', // 红色
  '#9f7aea', // 紫色
  '#ed64a6', // 粉色
  '#38b2ac', // 青色
  '#ecc94b', // 黄色
]
```

### 项目上下文 (ProjectContext)

```typescript
interface ProjectContext {
  staticContextIds?: string[]   // 静态上下文文件 ID 列表（多选）
  dynamicContextId?: string     // 动态上下文文件 ID
}
```

### 上下文类型说明

#### 静态上下文 (Static Context)

- **多选支持**：可以同时选择多个静态上下文文件
- **使用场景**：项目相关的背景资料、参考资料等
- **显示位置**：画布页面 Header 左侧
- **作用**：作为系统提示词的一部分发送给大模型

#### 动态上下文 (Dynamic Context)

- **单选限制**：每个项目只能关联一个动态上下文
- **使用场景**：临时性的上下文信息、拖拽收集的文本片段
- **显示位置**：画布页面 Header 右侧
- **快捷操作**：双击编辑、拖拽文字自动追加

### 节点作为上下文

- **选择方式**：点击节点头部的复选框
- **多选支持**：可以同时选择多个节点作为上下文
- **作用**：选中的节点内容（转写文本 + AI 回答）会作为对话历史发送给大模型

## 存储格式

### 项目列表文件

**路径**：`{userData}/projects/projects.json`

```json
{
  "projects": [
    {
      "id": "1234567890",
      "name": "我的项目",
      "createdAt": 1704067200000,
      "updatedAt": 1704153600000,
      "canvases": [
        {
          "id": "canvas-1",
          "type": "infinite",
          "viewport": { "x": 0, "y": 0, "zoom": 1 },
          "nodes": [
            {
              "id": "node-1234567890",
              "type": "voice-note",
              "position": { "x": 100, "y": 100 },
              "audioPath": "audio/node-1234567890.webm",
              "duration": 5000,
              "transcript": "这是转写文本",
              "transcriptStatus": "done",
              "agentResult": "这是 AI 回答",
              "agentStatus": "done",
              "selectedAsContext": false,

              "createdAt": 1704067200000

          ],

        "staticContextIds": ["context-1", "context-2"],

      }
    }
```

### 上下文文件存储

**路径**：`{userData}/contexts/contexts.json`

```json
{
  "contexts": [
    {
      "id": "context-1",
      "name": "项目背景",
      "type": "static",
      "color": "#4299e1",
      "content": "# 项目背景\n\n这是项目的背景资料...",
      "createdAt": 1704067200000,
      "updatedAt": 1704153600000
    },
    {
      "id": "context-3",
      "name": "动态上下文",
      "type": "dynamic",
      "color": "#66bb6a",
      "content": "临时收集的信息...",
      "createdAt": 1704067200000,
      "updatedAt": 1704153600000,
      "projectId": "1234567890"
    }
  ]
}
```

### 音频文件存储

**路径**：`{userData}/projects/{projectId}/audio/{nodeId}.{extension}`

- **格式**：webm 或 wav（可在设置中配置）
- **命名**：与节点 ID 对应
- **示例**：`audio/node-1234567890.webm`

## 文件组织

### 应用数据目录结构

```
{userData}/
├── projects/
│   ├── projects.json          # 项目列表
│   └── {projectId}/           # 项目专属目录
│       ├── audio/             # 音频文件目录
│       │   ├── node-xxx.webm
│       │   └── node-yyy.wav
│       └── ...                # 其他项目资源
│
└── contexts/
    └── contexts.json          # 上下文文件列表
```

### Electron API 接口

应用使用自定义的 `electronAPI` 进行文件操作：

```typescript
interface ElectronAPI {
  // 文件系统操作
  readFile(path: string, encoding: 'utf-8' | 'arraybuffer'): Promise<{ success: boolean; data?: any }>
  saveFile(path: string, content: string): Promise<void>
  saveFileBuffer(path: string, buffer: ArrayBuffer): Promise<void>
  
  // 目录操作
  mkdir(path: string): Promise<void>
  exists(path: string): Promise<boolean>
  
  // 应用路径
  getAppPath(name: 'userData' | 'appData' | ...): Promise<string>
}
```

## 数据流

### 创建语音笔记流程

```
用户长按画布
    ↓
开始录音 → 显示录音指示器
    ↓
用户释放鼠标
    ↓
停止录音 → 保存音频文件
    ↓
创建 CanvasNode（status: pending）
    ↓
自动转写（status: processing → done/error）
    ↓
自动 AI 回答（status: processing → done/error）
```

### AI 回答流程

```
触发条件：转写完成 / 文本编辑完成 / 重新生成
    ↓
收集上下文：
  - 静态上下文（多个文件内容合并）
  - 动态上下文（单个文件内容）
  - 选中的节点（转写文本 + AI 回答）
    ↓
构建消息列表（buildFullContextMessages）
    ↓
调用大模型 API（流式输出）
    ↓
实时更新节点 agentResult
    ↓
完成：agentStatus = 'done'
```

- 每个项目只有一个画布
- 使用 `canvas` 字段存储

## 注意事项

1. **数据备份**：`projects.json` 和 `contexts.json` 是核心数据文件，建议定期备份
2. **音频文件**：音频文件与节点 ID 绑定，删除节点时不会自动删除音频文件
3. **迁移兼容**：旧版本数据会自动迁移，但建议升级前备份数据
4. **存储限制**：应用数据存储在系统用户数据目录，注意磁盘空间
