## 已知问题

### 1. Web Audio API 格式限制
**问题描述**：
- Web Audio API 的 `decodeAudioData()` 不支持 WebM 格式
- 录音使用 MediaRecorder 默认生成 WebM/Opus 格式
- 需要转换为 WAV 才能被 Web Audio API 解码

**当前解决方案**：
- 使用文件路径直接传递给主进程
- 主进程使用 `sherpa-onnx.readWave()` 读取（支持更多格式）

**建议改进**：
- [ ] 在录音时直接生成 WAV 格式（如果浏览器支持）
- [ ] 或使用 ffmpeg 在后台转换音频格式
- [ ] 添加音频格式检测和自动转换机制

### 2. WASM 模块加载时间
**问题描述**：
- sherpa-onnx 使用 WebAssembly，首次加载需要时间
- 模型文件较大（~86MB），加载可能较慢

**当前解决方案**：
- 异步加载 WASM 模块
- 添加加载超时检测（5秒）

**建议改进**：
- [ ] 添加加载进度提示 UI
- [ ] 考虑在应用启动时预加载 WASM 模块
- [ ] 使用模型量化版本（INT8）减小体积

### 3. 模型文件管理
**问题描述**：
- 模型文件需要手动下载
- 模型路径在开发和生产环境不同
- 缺少模型版本管理

**当前解决方案**：
- 提供 `download-model.js` 脚本
- 自动检测开发和生产环境路径

**建议改进**：
- [ ] 自动下载模型（首次使用时）
- [ ] 模型版本检查和自动更新
- [ ] 支持多模型切换（小模型/大模型）
- [ ] 模型文件完整性校验

## 功能优化需求

### 高优先级

#### 1. 录音格式优化
```markdown
目标：解决 WebM 格式兼容性问题
方案：
1. 尝试使用 MediaRecorder 录制 WAV 格式（如果浏览器支持）
2. 或使用 MediaRecorder 录制 MP4/AAC 格式（Web Audio API 支持）
3. 在主进程使用 ffmpeg 转换 WebM 到 WAV

技术细节：
- 检查浏览器支持的 MIME 类型
- 优先使用非 WebM 格式
- 添加格式降级策略
```

#### 2. 加载状态提示
```markdown
目标：提升用户体验，避免等待焦虑
方案：
1. 添加 WASM 模块加载进度条
2. 显示模型加载状态
3. 首次使用引导（下载模型提示）

UI 设计：
- 启动画面显示加载进度
- 设置页面显示模型状态
- 录音按钮显示 ASR 就绪状态
```

#### 3. 错误处理优化
```markdown
目标：提供清晰的错误信息和解决方案
方案：
1. 模型文件缺失时提示下载命令
2. 音频格式不支持时提示转换方法
3. 网络问题（FunASR/Whisper）时提示切换本地识别

错误分类：
- 模型相关错误
- 音频格式错误
- 引擎初始化错误
- 识别过程错误
```

### 中优先级

#### 4. 多模型支持
```markdown
目标：支持不同场景下的模型选择
方案：
1. 小模型（~20MB）：快速响应，适合短语音
2. 大模型（~80MB）：高精度，适合长语音
3. 英语模型：支持英文识别

实现：
- 设置页面添加模型选择
- 自动根据语言切换模型
- 模型懒加载（按需加载）
```

#### 5. 实时语音识别
```markdown
目标：支持流式识别，边录边识别
方案：
1. 使用 sherpa-onnx 的 OnlineRecognizer
2. 分块发送音频数据
3. 实时显示识别结果

技术挑战：
- 需要重构录音逻辑
- 流式音频处理
- 结果合并和去重
```

#### 6. 热词支持
```markdown
目标：提高特定词汇的识别准确率
方案：
1. 支持自定义热词列表
2. 在设置页面配置热词
3. 热词权重调整

配置示例：
```json
{
  "hotwords": ["项目名称", "专业术语", "人名"],
  "hotwordsWeight": 1.5
}
```
```

### 低优先级

#### 7. 语音活动检测 (VAD)
```markdown
目标：自动检测语音开始和结束
方案：
1. 集成 sherpa-onnx VAD 功能
2. 自动停止录音（检测静音）
3. 过滤无效录音（纯静音）

优势：
- 节省存储空间
- 提高识别准确率
- 改善用户体验
```

#### 8. 语音合成 (TTS)
```markdown
目标：AI 回答可以语音播放
方案：
1. 集成 sherpa-onnx TTS 功能
2. 支持多种音色
3. 语速和音调调节

应用场景：
- 语音播报 AI 回答
- 辅助阅读
```

## 性能优化

### 1. 启动速度
```markdown
当前：WASM 模块 + 模型加载可能需要 2-5 秒
目标：减少到 1 秒以内
方案：
1. 延迟加载（首次录音时才初始化）
2. 预加载（应用启动后后台加载）
3. 模型压缩（使用 INT8 量化）
```

### 2. 识别速度
```markdown
当前：1秒音频约需 0.5-1 秒处理时间
目标：实时或更快
方案：
1. 使用 GPU 加速（CUDA/DirectML）
2. 调整线程数（根据 CPU 核心数）
3. 模型量化（INT8）
```

### 3. 内存占用
```markdown
当前：约 200-400MB
目标：减少到 100MB 以内
方案：
1. 使用更小的模型
2. 模型内存映射（mmap）
3. 识别完成后释放资源
```
